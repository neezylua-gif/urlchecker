# Как добавлять скам-ссылки в URL Guard Bot

Файл базы находится в корне проекта:

```text
scam_links.txt
```

Бот проверяет изменение этого файла перед каждой новой проверкой URL. После сохранения файла перезапуск бота не требуется.

## Самый простой вариант

Откройте `scam_links.txt` обычным текстовым редактором и добавляйте по одному адресу на строку:

```text
bad-domain.com
https://another-domain.com/fake-login
https://example.com/account/steal
```

Сохраните файл в кодировке UTF-8.

## Блокировка всего домена

```text
bad-domain.com
```

Будут заблокированы:

```text
https://bad-domain.com/
https://www.bad-domain.com/
https://login.bad-domain.com/account
```

То же самое можно записать явно:

```text
domain:bad-domain.com
*.bad-domain.com
```

## Блокировка конкретной ссылки

```text
https://example.com/fake-login
```

Если в записи нет query-параметров, бот также заблокирует варианты вроде:

```text
https://example.com/fake-login?id=123
https://example.com/fake-login?source=telegram
```

Если в базе query-параметры указаны, совпадение будет точным:

```text
url:https://example.com/verify?id=42
```

## Блокировка группы ссылок по началу адреса

```text
prefix:https://example.com/malware/
```

Будут заблокированы все адреса, начинающиеся так:

```text
https://example.com/malware/file.exe
https://example.com/malware/archive/payload.zip
```

Допустима запись со звёздочкой в конце:

```text
prefix:https://example.com/malware/*
```

## Комментарии

Пустые строки, а также строки, начинающиеся с `#` или `;`, не загружаются:

```text
# Фишинговые домены
bad-domain.com

; Вредоносные загрузки
prefix:https://example.com/downloads/
```

Не добавляйте комментарий в конце строки с URL, потому что символ `#` может быть частью ссылки.

## Docker

В `docker-compose.yml` файл уже подключён внутрь контейнера:

```yaml
volumes:
  - ./scam_links.txt:/app/scam_links.txt:ro
```

Редактируйте `scam_links.txt` на сервере рядом с `docker-compose.yml`. Пересборка контейнера и перезапуск не требуются.

## Ограничения и защита

- Максимальный размер базы по умолчанию — 1 МБ.
- Повторяющиеся записи автоматически убираются.
- Некорректные строки пропускаются и записываются в журнал.
- Если обновлённый файл невозможно прочитать, бот продолжает использовать последнюю успешно загруженную версию.
- Найденная в базе ссылка получает опасный вердикт до сетевого запроса.
- Цель редиректа также проверяется до перехода.

Путь и лимит размера можно изменить в `.env`:

```dotenv
SCAM_LINKS_FILE=scam_links.txt
SCAM_LINKS_MAX_BYTES=1048576
```
