# Security model

## Что блокируется

- Все адреса, для которых `ipaddress` не подтверждает публичную глобальную маршрутизацию.
- Loopback, RFC1918, link-local, metadata IP, shared `100.64.0.0/10`, multicast, reserved, unspecified и IPv4-mapped IPv6.
- Локальные имена: `localhost`, `.local`, `.internal`, `.lan`, `.home`, `.onion` и другие special-use suffixes.
- URL с `file:`, `ftp:`, `data:`, `gopher:` и любыми схемами, кроме HTTP/HTTPS.
- URL с `username:password@host`, управляющими/скрытыми символами и запрещёнными портами.
- Управляющие символы после percent-decoding, включая двойное кодирование (`%250d`).
- Редиректы на внутренние адреса, HTTPS→HTTP, циклы и междоменные переходы с чувствительными query-параметрами.
- HTML meta-refresh на опасные цели; проверяется только ограниченный префикс ответа.

## Почему DNS rebinding блокируется

HTTP-клиент использует собственный resolver. Все IP из DNS-ответа проверяются до передачи в `aiohttp`. DNS-кеш отключён, поэтому следующая попытка или редирект снова проходит проверку. Если хотя бы один адрес непубличный, соединение не создаётся.

## Защита от злоупотреблений

- Rate limit на Telegram-пользователя.
- Глобальный semaphore анализа.
- Лимит параллельных update-задач.
- Общий deadline анализа и отдельные сетевые таймауты.
- Не более двух одновременных соединений к одному хосту.
- Cookies, proxy environment и автоматическая распаковка отключены.
- Используется HEAD; GET ограничен Range и выполняется как fallback или для включённой проверки meta-refresh.

## Устойчивость анализа доменов

`tldextract` работает без сетевого обновления Public Suffix List. Если локальный snapshot или кеш недоступен, анализатор журналирует ошибку и использует базовый fallback по последним компонентам домена, не прерывая всю проверку.

## Meta-refresh

Парсятся только первые `META_REFRESH_MAX_BYTES` байт HTML, по умолчанию 2048. Найденная цель нормализуется и проходит все обычные проверки редиректа до сетевого обращения: схема, порт, DNS/SSRF, цикл, HTTPS downgrade и чувствительные query-параметры.

## Ограничение модели

Application-level SSRF protection должна дополняться сетевой политикой хостинга. В production рекомендуется отдельный контейнер/VM и firewall egress, запрещающий доступ к внутренним подсетям и metadata endpoint независимо от приложения.

## Локальная репутационная база

Файл `scam_links.txt` загружается с ограничением размера и перечитывается при изменении. Совпадение проверяется до исходящего соединения и до перехода по каждому HTTP/meta-refresh редиректу. Некорректные строки пропускаются, а при ошибке чтения сохраняется последняя успешно загруженная версия базы. В Docker файл подключается read-only bind mount.
